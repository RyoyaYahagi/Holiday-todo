# Holiday Todo App 要件定義書

## 1. 概要

### 1.1 プロダクト名
**Holiday Todo App**

### 1.2 プロダクトURL
https://holiday-todo-two-teal-11.vercel.app/

### 1.3 プロダクト概要
予定表（Googleカレンダーや.icsファイル）に基づいて、休日にタスクを自動でスマートにスケジューリングするTodoアプリケーション。不規則なシフト勤務の方でも、「次の休みにやること」を効率的に管理・消化できるよう設計されている。

### 1.4 開発目的
1. **AIエージェントの実験**: Google製のAIエージェント搭載IDE「Antigravity」を使えば、簡単にWebアプリを作成できるか検証
2. **タスクスケジュール自動化**: 知人が普段手動で行っている休日タスクのスケジュール作成を自動化

## 3. 機能要件

### 3.1 認証機能

#### FR-AUTH-001: Googleログイン
- **説明**: Googleアカウントを使用したOAuthログイン
- **詳細**:
  - Supabase Authを使用
  - ログイン後、データはクラウドに同期
  - 複数デバイス（PC・スマホ）間でデータを共有可能

#### FR-AUTH-002: ゲストログイン
- **説明**: アカウント登録なしでアプリを試用可能
- **詳細**:
  - Supabase匿名認証を使用
  - ブラウザセッション終了までデータを保持
  - ゲストからGoogleアカウントへの移行は非対応

---

### 3.2 タスク管理機能

#### FR-TASK-001: タスク作成
- **説明**: 新規タスクを作成する
- **詳細**:
  - タスクタイトル（必須）
  - スケジュールタイプの選択:
    - `priority`: 優先度ベース（自動スケジュール対象）
    - `time`: 日時を手動指定（自動スケジュール対象外）
    - `recurrence`: 繰り返しルール設定（自動スケジュール対象外）
    - `none`: 設定なし（プールに追加のみ）

#### FR-TASK-002: 優先度設定
- **説明**: タスクに優先度（P1〜P5）を設定
- **詳細**:
  - P5（最高）〜P1（最低）の5段階
  - 優先度の高いタスクから順に休日に自動割り当て
  - 設定画面で使用する最大優先度を変更可能（デフォルト: P5）

#### FR-TASK-003: 繰り返しタスク
- **説明**: 定期的に繰り返すタスクを設定
- **対応パターン**:
  - 毎日 (`daily`)
  - 毎週 (`weekly`) - 複数曜日選択可能
  - 隔週 (`biweekly`) - 複数曜日選択可能
  - 毎月 (`monthly`) - 日付指定
  - 毎年 (`yearly`)
  - 平日のみ (`weekdays`)
- **詳細**:
  - 繰り返し終了日の設定可能
  - 個別削除対応（その回だけ削除、今後の予定は残る）

#### FR-TASK-004: タスク編集
- **説明**: 既存タスクの内容を編集
- **詳細**:
  - タイトル、スケジュールタイプ、優先度、日時、繰り返し設定を変更可能

#### FR-TASK-005: タスク削除
- **説明**: タスクを削除
- **詳細**:
  - タスク削除時、関連するScheduledTaskも削除

#### FR-TASK-006: タスク完了
- **説明**: タスクを完了状態にする
- **詳細**:
  - スケジュール済みタスクの完了/未完了トグル
  - 完了タスクの一括削除機能

### 3.3 タスクリスト機能

#### FR-LIST-001: タスクリストの作成
- **説明**: タスクをグループ化するためのリストを作成
- **詳細**:
  - リスト名（必須）
  - カラー選択（8色のプリセットパレット）
  - デフォルトリストは削除不可

#### FR-LIST-002: タスクリストの編集・削除
- **説明**: リストの名前・色を変更、または削除

#### FR-LIST-003: タスクリストの並び替え
- **説明**: リストの表示順を上下に移動可能

#### FR-LIST-004: リストによるフィルタリング
- **説明**: 選択したリストに属するタスクのみ表示

---

### 3.4 自動スケジューリング機能

#### FR-SCHED-001: 休日への自動スケジュール
- **説明**: 優先度ベースのタスクを休日に自動割り当て
- **アルゴリズム**:
  1. 優先度順（P5→P1）にタスクをソート
  2. 休日を検索（イベントなし or「休み」イベントがある日）
  3. 1日あたりの最大タスク数まで割り当て
  4. 開始時間を考慮してスケジュール

#### FR-SCHED-002: 開始時間の自動調整
- **説明**: 前日の勤務状況に応じて開始時間を調整
- **詳細**:
  - 前日が「夜勤」→ 夜勤明けタスク開始時間を使用
  - それ以外 → 日勤後タスク開始時間を使用
  - 設定画面でカスタマイズ可能（0:00〜23:00）

#### FR-SCHED-003: スケジュール間隔設定
- **説明**: タスク間の間隔を設定
- **詳細**:
  - デフォルト: 2時間
  - 設定画面で変更可能

#### FR-SCHED-004: 1日の最大タスク数設定
- **説明**: 1日にスケジュールする最大タスク数を設定
- **詳細**:
  - デフォルト: 3件
  - 設定画面で変更可能

#### FR-SCHED-005: 休日の手動設定
- **説明**: カレンダーから特定日を手動でスケジュール対象/除外に設定
- **詳細**:
  - 「スケジュール対象」: 予定がある日でも強制的にスケジュール対象
  - 「スケジュール除外」: 休日でもスケジュール対象外

---

### 3.5 カレンダー連携機能

#### FR-CAL-001: Googleカレンダー同期
- **説明**: Googleカレンダーからイベントを取得
- **詳細**:
  - Google OAuth2.0認証
  - 過去1ヶ月〜今後1年のイベントを取得
  - イベント名から勤務タイプを自動判定:
    - 「夜勤」を含む → 夜勤
    - 「日勤」を含む → 日勤
    - 「休み」を含む → 休み

#### FR-CAL-002: ICSファイル読み込み
- **説明**: .icsファイルをアップロードしてイベントを取得
- **詳細**:
  - ファイル選択ダイアログまたはドラッグ＆ドロップ
  - パース結果をプレビュー表示

#### FR-CAL-003: カレンダー表示
- **説明**: 月間カレンダーでイベントとタスクを表示
- **詳細**:
  - 日付セルにイベントタイプ（日勤/夜勤/休み）を表示
  - スケジュール済みタスクをカレンダー上に表示
  - 休日（スケジュール可能日）をハイライト

#### FR-CAL-004: 日付詳細モーダル
- **説明**: 日付タップで詳細情報を表示
- **機能**:
  - その日のイベント一覧
  - スケジュール済みタスク一覧
  - タスクの追加・編集・削除
  - スケジュール対象/除外のトグル

### 3.6 通知機能

#### FR-NOTIF-001: LINE通知
- **説明**: LINE公式アカウント経由で通知を送信
- **詳細**:
  - QRコードで公式アカウントを友達追加
  - アプリ内でリンクコード（6桁）を入力して連携
  - 連携解除機能あり

#### FR-NOTIF-002: Discord通知
- **説明**: Discord Webhook経由で通知を送信
- **詳細**:
  - Webhook URLを設定

#### FR-NOTIF-003: 前日通知
- **説明**: 休日の前日夜にリマインド通知
- **詳細**:
  - 通知時刻を設定可能（デフォルト: 21:00）
  - ON/OFF切り替え可能

#### FR-NOTIF-004: タスク開始前通知
- **説明**: タスク開始直前にリマインド通知
- **詳細**:
  - 何分前に通知するか設定可能（デフォルト: 30分前）
  - ON/OFF切り替え可能

### 3.7 設定機能

#### FR-SET-001: スケジュール設定
- **設定項目**:
  - 最大優先度（1〜5、デフォルト: 5）
  - スケジュール間隔（時間単位）
  - 日勤後タスク開始時間（0:00〜23:00）
  - 夜勤明けタスク開始時間（0:00〜23:00）
  - 1日の最大タスク数

#### FR-SET-002: 通知設定
- **設定項目**:
  - 通知方法（LINE / Discord）
  - LINE連携設定
  - Discord Webhook URL
  - 前日通知のON/OFF・時刻
  - タスク開始前通知のON/OFF・分数

#### FR-SET-003: 外観設定
- **設定項目**:
  - テーマ（ライト / ダーク / システム設定）

#### FR-SET-004: データ管理
- **機能**:
  - データのエクスポート（JSON形式）
  - データのインポート（JSON形式）

## 5. 非機能要件

### 5.1 セキュリティ

#### NFR-SEC-001: Row Level Security（RLS）
- Supabase側で、ログインユーザー自身のデータにのみアクセスできるよう制限

#### NFR-SEC-002: 認証
- Supabase Auth（Google OAuth / Anonymous）を使用
- Google Calendar APIは本番環境での認証審査未実施のため、テストユーザー登録が必要

### 5.2 パフォーマンス

#### NFR-PERF-001: クラウド同期
- Supabase Realtimeによるリアルタイム同期
- TanStack React Queryによるキャッシュ管理

### 5.3 可用性

#### NFR-AVAIL-001: PWAオフライン対応
- Service Workerによる基本的なオフライン表示

### 5.4 互換性

#### NFR-COMPAT-001: ブラウザ対応
- モダンブラウザ（Chrome, Safari, Firefox, Edge）
- iOS Safari, Android Chrome
